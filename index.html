GitHub Hands-On Practice

Learn essential Git workflows by contributing to this repository .

Task 1: Direct Push to Main - Attendance List

In this first task, you'll add your name directly to the main branch's attendance list.

1 Clone the repository

Clone this repository to your local machine:

git clone https://github.com/oreste-abizera/git-hands-on-lab.git
cd git-hands-on-lab
2 Get the latest changes

Pull the latest changes from the main branch:

git pull origin main
3 Edit the attendance.html file

Open the attendance.html file in your preferred text editor and add your name to the list:

attendance.html
<li class="list-item">Your Name <span class="timestamp">(Added: Today's Date)</span></li>
Tip: Add your line right after the comment that says "Add your name below this line"
4 Commit your changes
git add attendance.html
git commit -m "Add [Your Name] to attendance list"
5 Push directly to main
git push origin main
Note: In this exercise, we're pushing directly to main for simplicity. In real-world projects, this practice is often restricted to maintain code quality.
6 Verify your changes

Check the repository on GitHub to confirm your name appears in the attendance list.

Task 2: Pull Request Workflow - Email List

For this second task, you'll add your email through a pull request workflow to practice code review.

1 Create a new branch

Create and switch to a new branch for your changes:

git checkout -b add-email-yourname
Tip: Replace "yourname" with your actual name, using lowercase and hyphens instead of spaces
2 Edit the emails.html file

Open the emails.html file and add your information to the table:

emails.html
<tr> <td>Your Name</td> <td>your.email@example.com</td> <td>your-github-username</td> </tr>
3 Commit your changes
git add emails.html
git commit -m "Add [Your Name]'s email to directory"
4 Push your branch
git push origin add-email-yourname
5 Create a Pull Request
  1. Go to the repository on GitHub
  2. Click on "Pull Requests" tab
  3. Click the "New pull request" button
  4. Select your branch in the "compare" dropdown
  5. Click "Create pull request"
  6. Add a descriptive title and comment
  7. Submit the pull request
Good PR title examples:
  • "Add Jane Doe's email to directory"
  • "Update email directory with my contact info"
6 Review a classmate's Pull Request

After submitting your PR, practice the code review process:

  1. Find an open pull request from a classmate
  2. Click on the "Files changed" tab to see their changes
  3. Review the code for any issues or improvements
  4. Add comments by clicking the "+" button on specific lines
  5. Submit your review (approve, request changes, or comment)
Review checklist:
  • Is the information correctly formatted?
  • Is the email in a valid format?
  • Did they add their information in the right place?
  • Did they accidentally modify or remove existing content?
